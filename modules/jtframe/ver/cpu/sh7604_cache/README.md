# SH7604 Cache Verilator Test

This directory contains a dedicated Verilator test bench for the SH7604 CPU cache path.

## Goal

The test exercises the `jtsh7604` wrapper against:

- `SH7604`
- `jtframe_cache_mux`
- `jtframe_burst_sdram`
- `modules/jtframe/verilator/sdram.cpp`

It is meant to stress the CPU cache and external memory bridge with:

- SDRAM boot code execution
- block copy and verify checks
- read-modify-write sequences
- partial byte and word writes
- longer sequential reads that force cache line refills

## Clocking

The simulation uses:

- `85.909090 MHz` for the Verilator top-level clock
- alternating `CE_R` and `CE_F` every top-level clock, with no blank master
  clock cycles between CPU phases

The `CE_R` and `CE_F` pulses are single-clock, non-overlapping phases. After
reset they run as `CE_R`, `CE_F`, `CE_R`, `CE_F`, so the test stresses the CPU
with consecutive phase enables instead of the normal sparse 25 MHz timing.

## Boot Image

The CPU boots from `sdram_bank0.bin`, which is generated by:

- `modules/jtframe/ver/cpu/sh7604_cache/sh7604_bank0.s`
  (assembled with `sh-elf-as`)

That image contains:

- the SH-2 test program
- source and destination test data
- cache fill scan data

## Running

From this directory:

```bash
bash ./sim.sh --keep
```

`--keep` enables FST tracing and keeps `test.fst`.

All extra arguments are forwarded to the simulation binary.
